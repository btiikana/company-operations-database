# Company Operations Database

This repository documents a project-based database engineering learning path focused on designing a company operations database system.

The project is built around the Island Supply Company Operations Database System, a simulated business database used to practice business analysis, database design, SQL, PostgreSQL, technical documentation, and later Python automation.

## Project Goal

The goal of this project is to learn how to move from business requirements to a structured relational database system.

The project will include:

- Business process analysis
- Stakeholder and requirements documentation
- Data modeling
- Relational table planning
- PostgreSQL schema design
- SQL queries and reports
- Data integrity constraints
- Transactions and concurrency practice
- Performance and indexing notes
- Security and permissions documentation
- Backup and recovery documentation
- Python automation scripts

## Repository Structure

- `docs/` - project documentation, requirements, business rules, design notes, and guides
- `diagrams/` - process diagrams, ERDs, and visual models
- `sql/` - SQL scripts for schema creation, seed data, queries, constraints, reports, and tests
- `data/` - sample CSV data
- `python/` - Python scripts for loading data, reports, alerts, and automation
- `progress/` - learning path, course tracking, piece tracking, and weekly logs
- `certificates/` - course certificates and related proof of learning
- `backups/` - backup and restore examples

## Build Approach

This project follows a piece-by-piece learning path. Each piece creates one practical artifact that becomes part of the final database system.

The main build flow is:

Business process understanding  
-> business rules  
-> data modeling  
-> table planning  
-> PostgreSQL implementation  
-> SQL queries and reports  
-> constraints and transactions  
-> administration, security, backup, and automation
